Apache Virtual Hosts – Beginner Training & Hands-on Lab

Target Audience: Beginners, Linux users, DevOps aspirants
OS: Ubuntu 20.04 / 22.04 / 24.04
Web Server: Apache 2.4

1. What is a Virtual Host?

A Virtual Host allows Apache to host multiple websites on a single server. Each website can have:

  • Its own domain name
  • Its own folder
  • Its own logs
Example:
site1.local → /var/www/site1.local/
site2.local → /var/www/site2.local/

2. Why Virtual Hosts Are Needed

  • Shared hosting
  • Development & testing
  • Multiple projects on one server
  • Cost saving

3. Apache Folder Structure (Ubuntu)

/etc/apache2/sites-available/   → Virtual Host configs
/etc/apache2/sites-enabled/     → Enabled sites (symlinks)
/var/www/                       → Website files
/var/log/apache2/               → Logs

4. LAB 1 – Create Your First Virtual Host

Objective

Create and access:

http://site1.local

Step 1: Install Apache

sudo apt update
sudo apt install apache2 -y

Step 2: Create Website Directory

sudo mkdir -p /var/www/site1.local/public_html
sudo chown -R $USER:$USER /var/www/site1.local
sudo chmod -R 755 /var/www

Step 3: Create Test Web Page

nano /var/www/site1.local/public_html/index.html

<h1>Site1 Virtual Host Working</h1>

Step 4: Create Virtual Host Configuration

sudo nano /etc/apache2/sites-available/site1.local.conf

<VirtualHost *:80>
    ServerName site1.local
    ServerAlias www.site1.local
    DocumentRoot /var/www/site1.local/public_html

    <Directory /var/www/site1.local/public_html>
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/site1_error.log
    CustomLog ${APACHE_LOG_DIR}/site1_access.log combined
</VirtualHost>

Step 5: Enable Site

sudo a2ensite site1.local.conf
sudo a2dissite 000-default.conf

Step 6: Test & Reload Apache

sudo apache2ctl configtest
sudo systemctl reload apache2

Step 7: Update Hosts File

sudo nano /etc/hosts

127.0.0.1 site1.local

Validation

Open browser and visit:

http://site1.local

5. LAB 2 – Multiple Virtual Hosts

Objective

Run two websites on same Apache server

Create Second Site

sudo mkdir -p /var/www/site2.local/public_html
nano /var/www/site2.local/public_html/index.html

<h1>Site2 Virtual Host Working</h1>

Create Config

sudo nano /etc/apache2/sites-available/site2.local.conf

<VirtualHost *:80>
    ServerName site2.local
    DocumentRoot /var/www/site2.local/public_html

    <Directory /var/www/site2.local/public_html>
        Require all granted
    </Directory>
</VirtualHost>

Enable and Reload

sudo a2ensite site2.local.conf
sudo systemctl reload apache2

Update Hosts

127.0.0.1 site1.local
127.0.0.1 site2.local

6. Logs & Troubleshooting

Log Files

/var/log/apache2/site1_access.log
/var/log/apache2/site1_error.log

Common Errors

  • 403 Forbidden → Permission issue
  • Default page shown → Default site not disabled
  • Apache fails → Syntax error

7. Quick Revision

  • Virtual Hosts allow multiple sites
  • Each site has its own config
  • a2ensite enables site
  • Always run configtest
